### 1. 初始化仓库的作用是什么？
执行 `git init` 会在当前目录创建一个 `.git` 隐藏文件夹，初始化一个空的 Git 仓库。它的核心作用是让当前目录被 Git 追踪，开启文件的版本控制功能，为后续记录修改、保存版本做准备。

### 2. 保存版本记录前，为什么需要先选择要保存的修改？
Git 采用“暂存区”机制，`git add` 可以选择性地将部分修改加入暂存区。这样做的好处是：
- 可以分批次、分功能提交修改，避免一次提交大量无关变更；
- 让每次提交都聚焦于一个明确的目标，提交记录更清晰；
- 避免误提交临时文件或未完成的代码。

### 3. 提交说明为什么不能随便写？
提交说明（`commit message`）是版本记录的“说明书”，清晰的提交说明有以下作用：
- 方便自己和团队快速了解每次提交做了什么修改；
- 在需要回退版本、排查问题时，能快速定位关键提交；
- 是项目维护和协作的重要依据，规范的提交说明能提升项目的可维护性。

### 4. 工作区、暂存区、本地仓库分别表示什么？
- **工作区**：你电脑上实际存放文件的目录，也就是你直接编辑文件的地方。
- **暂存区（Stage）**：Git 提供的临时缓冲区，通过 `git add` 可以将工作区的修改存入这里，等待提交。
- **本地仓库（Repository）**：Git 永久保存所有版本记录的地方，通过 `git commit` 可以将暂存区的修改提交到这里，生成新的版本记录。

### 5. 查看仓库状态有什么作用？
通过 `git status` 查看仓库状态，可以：
- 了解文件当前处于“未跟踪/已修改/已暂存”的哪个阶段；
- 避免误提交未暂存的修改，或遗漏需要提交的文件；
- 清晰掌握 Git 仓库的工作进度，减少操作失误。
